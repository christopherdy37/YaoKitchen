# Yao's Kitchen — Wake Catering Platform

Premium wake catering service based in Metro Manila, Philippines. Clients inquire via the website; owner confirms bookings through Viber.

## Tech Stack

| Layer | Choice |
|---|---|
| Framework | Next.js 14 (App Router) |
| Language | TypeScript |
| Styling | Tailwind CSS + shadcn/ui |
| Animation | Framer Motion |
| ORM | Prisma 7 |
| Database | PostgreSQL (Railway) |
| Auth | iron-session (cookie-based, admin only) |
| Forms | react-hook-form + zod + @hookform/resolvers |
| Date picker | react-day-picker |
| QR codes | qrcode |

## Folder Structure

```
/

│   ├── seed.ts             # Admin user bootstrap
│   └── migrations/         # Migration history
├── middleware.ts            # Protects /admin/* except /admin/login
├── prisma.config.ts         # Prisma 7 config (loads DATABASE_URL)

```

## Database Models

- **Inquiry** — customer booking request; holds contact info, event details, package, status
- **BlockedDate** — dates owner has marked unavailable
- **Partner** — funeral homes / partner venues with unique slugs for referral tracking
- **AdminUser** — admin login credentials (bcrypt hashed)

## Environment Variables

| Variable | Purpose |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string (Railway) |
| `SESSION_SECRET` | iron-session encryption key (32+ chars) |
| `NEXT_PUBLIC_VIBER_NUMBER` | Viber number shown on public pages |
| `ADMIN_SEED_EMAIL` | Email for initial admin account (seed only) |
| `ADMIN_SEED_PASSWORD` | Password for initial admin account (seed only) |

## Booking Reference Format

`YK-YYYYMMDD-XXXX` — generated by `generateBookingRef()` in `lib/utils.ts`.
Example: `YK-20260428-A3F9`

## Key Conventions

- All admin routes live under `/admin/*`; middleware redirects unauthenticated users to `/admin/login`
- Prisma client imported from `@/lib/prisma` (singleton safe for Next.js hot-reload)
- Session read via `getSession()` from `@/lib/session` in server components / route handlers
- Public inquiry source defaults to `"website"`; partner referrals set `source = "partner"` and `partnerSlug`

## npm Scripts

```bash
npm run dev          # Start dev server
npm run db:migrate   # Run Prisma migrations (dev)
npm run db:generate  # Regenerate Prisma client
npm run db:seed      # Seed first admin user
```

## Deploy (Railway)

1. Create PostgreSQL service on Railway, copy `DATABASE_URL`
2. Set all env vars in Railway project settings
3. Push to GitHub — Railway auto-deploys via `railway.toml`
4. On first deploy, run `npm run db:seed` from Railway shell
